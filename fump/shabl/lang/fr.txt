[T_SENDING_TIPS]	
	<h2>Astuces pour configurer les headers pour les emails retournés. </h2>
                    <p>Si vous avez l’intention à travailler avec les messages retournés, vous pouvez définir ici les options d’envoi.</p>
                    <p>&nbsp;</p>
                    <p><font color="#FF0000"><strong>Attention!</strong></font> 
                      Messages non envoyés seront retournés selon la configuration par défaut, 
                      Vous avez besoin de définir ou vérifier les options pour les messages non envoyés, si vous ne pouvez pas trouver le messages retournés dans votre boîte email ou vous avez besoin de spécifier une autre boîte email, différente de celle que vous utilisez dans vos mailings pour « From » ou « Répondre à » .</p>
                    <p>Habituellement, tous les messages qui n’ont pas pu être distribués, seront retournés à l’email que vous avez indiqué dans le header Retour-path dans vos messages. Vous avez plusieurs solutions pour effectuer cela.</p>
                    <p><strong><u>Envoi SMTP</u></strong></p>
                    <p>Si vous utilisez SMTP pour envoyer vos messages, vous pouvez spécifier 
                      Un différent email de votre headers 'DE' ou 'Répondre à'. 
                      Vous avez besoin de cocher la case'Utiliser un email suivant pour l’envoi SMTP' 
                      Et spécifier email. <br>
                      <br>
                      <strong><u>Configuration pour l’utilisation de la commande Sendmail</u></strong></p>
                    <p>Peut-être, les résultats ne seront pas optimaux, mais vous avez quelques bonnes alternatives: </p>
                    <p><br>
                      <strong><em>Technique 1:</em></strong> Un -f flag</p>
                    <p>Vous avez besoin de cocher 'Utiliser sendmail -f flag' et spécifier un email 
                      ici</p>
                    <p>Cela permet de configuer le header de Path-Retour.</p>
                    <p>&nbsp;</p>
                    <p><em><strong>Technique 2: </strong></em>Retour-Path header</p>
                    <p>Si cela ne marche pas, vous pouvez essayer de configurer le header du Path-Retour ouvertement.  Ce n’est pas le meilleur moyen, mais Qmail permet de faire cela.</p>
                    <p>&nbsp;</p>
                    <p><strong>Technique 3: </strong>”Erreurs-à” header</p>
                    <p>Un “Erreurs-à” header semble d’avoir être créé uniquement pour cette tâche, mais actuellement c’est un header désapprouvé. Utilisez-le avec précaution. Pour qu’il puisse effectuer le travail, vous devez configurer Sendmail pour voir un header actuel « Erreurs- à ». Si les techniques précédentes ont échoué, vous pouvez essayer de spécifier le header « Erreurs-à ».</p>
                    <p>&nbsp;</p>
                    <p>Pour tester une de ces configurations, envoyez-vous un teste message et regarder la source de ce message même dans votre logiciel de la réceptions des emails. Dans votre mail headers, vous devez apercevoir le header de Path-Retour, comme: </p>
                    <blockquote>
                      <p><code>Retour-Path: &lt;info-store@sellwide.net&gt;<br>
                        Envoyé à: helpdesk@sellwide.net<br>
                        Contenu-Disposition: en ligne<br>
