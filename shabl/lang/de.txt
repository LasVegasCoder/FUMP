[T_SENDING_TIPS]
	<h2>Infos für Einstellungen u. Steuerung der Bounced-Mail-Header. </h2>
                    <p>Zur Bearbeitung von Bounced-Meldung können sie hier die Einstellung festlegen. </p>
                    <p>&nbsp;</p>
                    <p><font color="#FF0000"><strong>ACHTUNG!</strong></font>
                      Bounced-Nachrichten (Rücklauf-Nachrichten Sendefehler) können sie mit Einstellungen steuern. Sie müssen hier die Optionen definieren um die Nachrichten an die gewollte Mailbox zu senden, wobei die Einstellung ihrer FROM oder Reply-To Vorsätze unterschiedlich sein muß. Viel Erfolg!<br><br>
<p>Normalerweise gehen alle Rückmeldungen an die Emailadresse die als Reply-To eingetragen ist zurück.  Nutzen sie hier die Optionen um dies zu verändern.</p>
                    <p><strong><u>Mit SMTP senden</u></strong></p>
                    <p>Wenn Sie SMTP benutzen, um Ihre Emailnachrichten zu senden, können die Einträge  'FROM' oder 'Reply-To' eine spezielle Emailadresse zuordnen. Dazu müssen Sie die Checkbox vor 'Folgende Mailadresse als SMTP-Absender nutzen' aktivieren und die gewünschte Emailadresse eintragen.   <br>
                      <br>
                      <strong><u>Einstellungen zur Nutzung von Sendmail</u></strong></p>
                    <p>Ihre Resultate können mit verschiedenen Methoden (Optionen) gemischt werden. Testen sie verschiedene Variationen: </p>
                    <p><br>
                      <strong><em>Option 1:</em></strong> Sendmail-Anhang -f </p>
                    <p>Aktivieren sie die Checkbox vor 'Sendmailzusatz -f an folgende Emailadresse' und                                     tragen sie die gewünschte Emailadresse ein.</p>
                    <p>Damit legen sie den 'Return-Path' fest.</p>
                    <p>&nbsp;</p>
                    <p><em><strong>Option 2: </strong></em>Return-Path header</p>
                    <p>Wenn das nicht arbeitet, können Sie versuchen, den Return-Path Header direkt einzustellen.  Dieses ist nicht die beste Weise, aber Qmail erlaubt Ihnen, dies zu tun.</p>
                    <p>&nbsp;</p>
                    <p><strong>Option 3: </strong>Errors-To header</p>
                    <p>Der 'Errors-To' Header scheint, gerade für diese Aufgabe hergestellt worden zu sein, aber es ist wirklich ein mißbilligter Eintrag, also Gebrauch mit Vorsicht.  Damit es funktioniert, müssen Sie Sendmail konfigurieren, um den 'Errors-To Header' wirklich zu sehen. Wir empfehlen nicht, diesen Eintrag zu verwenden, aber wenn alles andere nicht funktioniert, können Sie diesen Fehlervorsatz spezifizieren. </p>
                    <p>&nbsp;</p>
                    <p>Um ihre Konfigurationen zu prüfen, senden Sie sich eine Testmail und prüfen den Quelltext der Nachricht in ihrem Mailprogramm.  Im Mail-Header können sie dann den  Return-Pfad sehen:</p>
                    <blockquote>
                      <p><code>Return-Path: &lt;info-store@sellwide.net&gt;<br>
                        Delivered-To: helpdesk@sellwide.net<br>
                        Content-Disposition: inline<br>
                        Content-Transfer-Encoding: binary<br>
                        Content-Type: text/plain<br>
                        MIME-Version: 1.0<br>
                        X-Mailer: MIME::Lite 2.117 (F2.71)<br>
                        Date: Tue, 14 Jun 2005 15:26:41 UT<br>
                        From: FUMP Registration &lt;info-store@sellwide.net&gt;<br>
                        To: Konstantin Ursaloff &lt;info-store@sellwide.net&gt;<br>
                        Subject: User mom_id2000@yahoo.com registered at the dlfumlpl
                        account </code></p>
                    </blockquote>

[T_BOUNCE_HELP]
<p>Hier können sie POP3 Accounts für aufgelaufene Bounce-Mails einrichten und bearbeiten.
                    Ebenfalls werden Regeln für die Behandlung und Aktionen von Bounced-Mails und den  Usern eingestellt (z. Bsp. löschen oder User deaktivieren):<br>
                    <br>
                    <strong>POP3 Server</strong> - ihr POP3 Server für die Emailnachrichten</p>
                  <p><strong>POP3 Port</strong> - standard POP3 Port ist 110</p>
                  <p><strong>POP3 User</strong> - Username zum Zugriff auf die Mailbox</p>
                  <p><strong>POP3 Passwort</strong> - Passwort zum Zugriff auf die Mailbox<br>
                  </p>
                  <p><strong>Lösche&nbsp;User&nbsp;nach</strong>:<br>
                    - <strong>hard bounces:</strong> Hier ist die Anzahl wieviel der hard bounced
                    Nachrichten angenommen werden, bevor der User gelöscht oder deaktiviert wird. (Hard
                    bounces sind permanente Fehler die entstehen weil der Host nicht gefunden wird,
                    die Nachricht durch SPAM-Filter zurückgewiesen wird, dies bedeutet dann auch das ihre Nachricht nicht ausgeliert werden kann). Die Grundeinstellung erlaubt 1 hard bounced Nachricht bevor gelöscht/deaktiviert wird. Diese Einstellung können sie jederzeit ändern.</p>
                  <p>- <strong>soft bounces:</strong> Hier ist die Anzahl wieviel der soft bounced Nachrichten zugelassen werden bevor
                    der User gelöscht oder deaktiviert wird. (Soft bounces
                    sind temporäre Fehler die entstehen wenn zum Beispiel die Mailbox des Empfängers voll ist (over quota)
                    etc). Die Grundeinstellung erlaubt 3 soft bounced Nachrichten bevor gelöscht/deaktiviert wird. Diese Einstellung können sie jederzeit ändern.</p>
                  <p><strong><br>
                    Aktionen: </strong></p>
                  <p>- <strong>User aus Datenbank löschen:</strong> Der User wird endgültig aus der Datenbank gelöscht.</p>
                  <p>- <strong>Status inaktiv setzen:</strong> Der User wird deaktiviert</p>
                  <p>- <strong>User in die BlackList eintragen:</strong>
                    Ist diese Option gewählt werden User automatisch in die BlackListe eingetragen wenn sie gelöscht oder deaktiviert werden.</p>
